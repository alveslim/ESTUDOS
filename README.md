# 🚀 Trilha de Estudos em Python, Web e DevOps

Este repositório é o registro da minha jornada de aprendizado contínuo em desenvolvimento de software. Aqui documento códigos, projetos práticos e anotações teóricas, cobrindo desde os fundamentos da programação orientada a objetos até a conteinerização de aplicações.

## 🛠️ Tecnologias e Ferramentas
* **Linguagem:** Python
* **Web Framework:** Flask
* **Banco de Dados:** SQLite, manipulação via ORM
* **Infraestrutura e DevOps:** Docker, WSL (Subsistema Windows para Linux)
* **Versionamento:** Git e GitHub

---

## 📚 Estrutura do Repositório (Módulos)

O repositório está organizado de forma evolutiva, acompanhando a complexidade dos tópicos abordados:

* **Módulo 1 ao 3 - Fundamentos e POO:** Estruturas básicas de Python, importação de módulos e Programação Orientada a Objetos.
* **Módulo 4 - Manipulação de Arquivos:** Leitura e escrita de arquivos `.txt` e `.csv`, manipulação de dicionários e ordenação de dados.
* **Módulo 5 - Banco de Dados (BD):** Integração do Python com SQLite, operações CRUD, uso de ORM e execução de queries em tabelas.
* **Módulo 6 - Automações:** Scripts práticos para o dia a dia, como manipulação de caminhos (paths), renomeação em lote, organização de diretórios e envio automatizado de e-mails.
* **Módulo 7 - Flask:** Criação de aplicações web utilizando o microframework Flask.
* **Docker:** Configuração de ambientes isolados, criação de imagens (`Dockerfile`, `.dockerignore`) e gerenciamento de containers da aplicação.
* **Git e GitHub:** Anotações sobre repositórios locais e remotos, branches e comandos essenciais de versionamento (como `.gitignore`).

---

## 🐳 Foco Recente: Conteinerização com Docker

Atualmente, o foco dos estudos está na conteinerização das aplicações em Python para garantir consistência entre ambientes. Alguns dos comandos e conceitos dominados recentemente incluem:

* Configuração do ambiente Linux no Windows via `wsl --install`.
* Busca e execução de imagens base (`docker pull`, `docker run`).
* Gerenciamento de containers ativos e inativos (`docker ps`, `docker ps -a`, `docker rm`).
* Construção de imagens customizadas com `docker build --tag <nome> .`.
* Acesso interativo ao terminal de containers Linux rodando em background com `docker exec -it <container_id> bash`.
